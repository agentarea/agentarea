üìä Quick Start

The MCP Manager provides APIs for managing Model Context Protocol server instances. You can create, monitor, and delete MCP instances through HTTP endpoints.

Current Service

http://localhost:7999

Base URL for all API calls

Authentication

No authentication required for this deployment

üîç Core Endpoints

GET /health
Check service health and status
Try It
GET /instances
List all MCP instances
Try It
POST /instances
Create a new MCP instance
curl -X POST http://localhost:7999/instances \ -H "Content-Type: application/json" \ -d '{ "instance_id": "my-mcp-server", "name": "My MCP Server", "service_name": "my-service", "image": "nginx:alpine", "port": 80, "workspace_id": "test-workspace" }'
GET /instances/{id}
Get details of a specific instance
DELETE /instances/{id}
Delete an MCP instance
GET /instances/{id}/health
Check health of a specific instance

üîß Instance Management Examples

Create an NGINX MCP Instance

curl -X POST http://localhost:7999/instances \ -H "Content-Type: application/json" \ -d '{ "instance_id": "nginx-demo", "name": "NGINX Demo Server", "service_name": "nginx-demo", "image": "nginx:alpine", "port": 80, "environment": { "ENV": "demo" }, "workspace_id": "demo-workspace", "resources": { "requests": { "cpu": "100m", "memory": "128Mi" }, "limits": { "cpu": "200m", "memory": "256Mi" } } }'

Check Instance Health

curl http://localhost:7999/instances/nginx-demo/health

Access Instance via Proxy

# Get the proxy URL from instance details curl http://localhost:7999/instances/nginx-demo # Access via proxy (URL from above response) curl http://localhost:7999/mcp/nginx-demo-abc123/

Delete Instance

curl -X DELETE http://localhost:7999/instances/nginx-demo
